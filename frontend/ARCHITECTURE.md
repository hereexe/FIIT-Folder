# Архитектура Frontend (FIIT-Folder)

Этот документ описывает архитектуру клиентской части приложения, расположенной в директории `react_ts`.

## Технологический стек
*   **Framework**: React 18
*   **Language**: TypeScript
*   **Build Tool**: Vite
*   **Styling**: Tailwind CSS
*   **Package Manager**: npm / pnpm

## Методология: Feature-Sliced Design (FSD)

Проект организован в соответствии с методологией [Feature-Sliced Design](https://feature-sliced.design/).
Код разбит на слои (Layers), слайсы (Slices) и сегменты (Segments).

### Структура слоев (`src/`)

Слои расположены в порядке от "грязного" (высокая связность) к "чистому" (низкая связность). Верхние слои могут использовать нижние, но не наоборот.

1.  **app/** (Application Layer)
    *   Инициализация приложения.
    *   `providers/`: Провайдеры контекста (Theme, Auth и т.д.).
    *   `router/`: Настройка маршрутизации (React Router).
    *   `styles/`: Глобальные стили и подключение Tailwind.

2.  **pages/** (Pages Layer)
    *   Компоненты полных страниц. Собираются из виджетов, фич и сущностей.
    *   `main`: Главная страница.
    *   `docs`: Страница документации.
    *   `file-view`: Просмотр файла.
    *   `exam-type`: Страница выбора типа экзамена/материала.

3.  **widgets/** (Widgets Layer)
    *   Самостоятельные UI-блоки, объединяющие фичи и сущности.
    *   `header`: Шапка сайта.
    *   `header-layout`: Лейаут с шапкой.

4.  **features/** (Features Layer)
    *   Обрабатываемые пользовательские сценарии (действия, несущие бизнес-ценность).
    *   `file-upload`: Логика и UI загрузки файлов.
    *   `search`: Поиск материалов.

5.  **entities/** (Entities Layer)
    *   Бизнес-сущности. Содержат UI-компоненты (карточки, строки) и простую логику отображения.
    *   `material`: Учебный материал.
    *   `file`: Файл.
    *   `exam`: Экзамен/предмет.

6.  **shared/** (Shared Layer)
    *   Переиспользуемый код, не привязанный к специфике бизнеса.
    *   `ui/`: UI Kit (кнопки, инпуты, модалки).
    *   `api/`: Базовый HTTP-клиент и настройки API.
    *   `hooks/`: Общие хуки.
    *   `lib/`: Утилиты и хелперы.

## Правила импортов
*   Модуль может импортировать только модули из **нижележащих** слоев.
*   Пример: `features` может использовать `entities` и `shared`, но не `widgets` или `pages`.
*   Импорты внутри одного слайса разрешены.
*   Перекрестные импорты между слайсами одного слоя **запрещены** (High Cohesion, Low Coupling).

## Расположение файлов
Основной код находится в `frontend/react_ts/src`.
В корне `frontend/` также находится папка `api/`, содержащая, предположительно, генерируемые или общие типы API.
